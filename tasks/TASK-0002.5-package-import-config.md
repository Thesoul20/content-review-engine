# TASK-0002.5: Normalize Package Import Configuration

## Goal

Fix the package import configuration so tests can import `content_review_engine` without manually modifying `sys.path`.

This is a project hygiene task before adding more modules.

---

## Background

After TASK-0002, tests pass, but `tests/test_models.py` uses a manual `sys.path` append to import from the `src/` layout.

This should be fixed at the project configuration level.

---

## Scope

Fix Python package import configuration for the `src/` layout.

Allowed changes:

- `pyproject.toml`
- `tests/test_models.py`
- possibly test configuration files if needed
- `PROJECT_STATE.md`
- `CHANGELOG.md`

---

## Do Not Modify

Do not add:

- CLI code
- MCP code
- Skill files
- FastAPI code
- PydanticAI code
- Markdown parser
- Review rules

Do not change:

- `ReviewIssue`
- `ReviewResult`
- `ReviewProfile`

---

## Acceptance Criteria

- `tests/test_models.py` no longer manually modifies `sys.path`.
- `uv run pytest` passes.
- The package can be imported normally:

```python
from content_review_engine.core.models import ReviewIssue
```
